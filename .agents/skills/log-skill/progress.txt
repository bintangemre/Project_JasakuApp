## 2026-07-07 — Identity Verification System (OCR KTP + Liveness + Face Matching)

### Arsitektur

Sistem verifikasi identitas provider terdiri dari 4 prioritas yang diimplementasikan berurutan:
1. **Tabel identity_verifications** — menyimpan hasil OCR, face match, liveness
2. **OCR KTP on-device** — ML Kit Text Recognition (Latin) untuk baca NIK/Nama/Alamat dari foto KTP
3. **Face matching** — InsightFace Python microservice untuk bandingkan face embedding KTP vs selfie
4. **Liveness detection** — 3 challenge: kedip → senyum → miringkan kepala via ML Kit Face Detection

### Backend (`jasaku-backend/`)

- **Modified** `prisma/schema.prisma`:
  - Tambah model `identity_verifications` dengan fields: `nik`, `ocr_full_name`, `ocr_birth_place`, `ocr_birth_date`, `ocr_address`, `ocr_raw_result` (Json), `face_match_score`, `face_match_status`, `liveness_data` (Json), `liveness_status`, `verified_at`
  - Relation ke `provider_profiles` (one-to-one, onDelete Cascade)
  - Index on `provider_id`
  - Tambah relasi balik `identity_verifications?` di `provider_profiles`

- **Modified** `src/modules/auth/auth.service.ts`:

  - Di dalam transaksi, buat `identity_verifications` record setelah profile, isi OCR fields + `liveness_status` (passed/failed berdasarkan completed challenges)
  - Setelah transaksi, panggil `runFaceMatchAsync()` non-blocking jika KTP & selfie tersedia
